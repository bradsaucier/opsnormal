# Support

## Scope

OpsNormal is intentionally narrow.
Support stays inside the local-only product boundary documented in the README, ADRs, and SECURITY.md.

## Where to ask for help

- Use the bug report template for reproducible defects in the current product scope
- Use the feature request template for scoped improvements that preserve the repo boundary
- If GitHub Discussions is enabled for the repository, use Discussions for open-ended usage questions that do not map cleanly to a code change

## Security issues

Do not open public issues for vulnerabilities.

Use GitHub private vulnerability reporting:

- https://github.com/bradsaucier/opsnormal/security/advisories/new

## Response expectations

- This is a solo-maintained project
- There is no guaranteed response time
- Release-blocking defects, data-integrity risks, and security issues take priority over feature work

## Funding posture

This repository does not accept sponsorships, donations, or paid support.
Keep GitHub funding surfaces disabled for this project.
